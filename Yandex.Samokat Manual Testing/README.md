### 📝 Краткое описание проекта

Первая часть дипломного проекта посвящена ручному тестированию веб-приложения и проектированию тестов для мобильного приложения «Яндекс Самокат». Выполнены: анализ требований, составление чек-листа для экрана «Статус заказа», таблицы проверок валидации полей для экрана «Сделать заказ», тест-кейсов для функциональности мобильного приложения, а также заведены баг-репорты на найденные дефекты.

### 🛠 Использованные навыки

- Анализ требований и макетов (Figma)
- Составление чек-листов на основе требований
- Проектирование проверок валидации полей ввода
- Написание тест-кейсов (предусловия, шаги, ожидаемый результат)
- Проведение ручного функционального тестирования веб-приложения
- Составление баг-репортов с приоритетами и статусами
- Работа с Google Таблицами для документирования результатов
- Отслеживание статусов тестов (PASSED/FAILED) и привязка к багам

### 📌 Вывод

Проект позволил применить полный цикл ручного тестирования: от изучения требований и макетов до составления тестовой документации и регистрации дефектов. Были выявлены некорректные сценарии валидации, баги в логике работы экрана «Статус заказа» и проблемы вёрстки мобильного приложения. Этот этап стал основой для дальнейшей автоматизации, показав важность тщательного проектирования тестов перед написанием автотестов.
