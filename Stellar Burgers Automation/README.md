### 📝 Краткое описание проекта

Вторая часть дипломного проекта включает автоматизацию тестирования сервиса Stellar Burgers на трёх уровнях: юнит-тесты для класса Burger (Java, JUnit, Mockito), API-тесты для эндпоинтов создания пользователя, логина и заказов (RestAssured), UI-тесты для проверки регистрации, входа и работы конструктора (Selenium WebDriver, Page Object). Все тесты сопровождаются генерацией отчётов Allure и JaCoCo, а также кросс-браузерным запуском (Google Chrome и Яндекс Браузер).

### 🛠 Использованные навыки

- Написание юнит-тестов с использованием JUnit 4, Mockito (моки, стабы, параметризация)
- Оценка покрытия кода с помощью JaCoCo (100% для класса Burger)
- Автоматизация API-тестов с RestAssured (позитивные и негативные сценарии, проверка кодов и тела ответа)
- UI-автоматизация с Selenium WebDriver (Page Object pattern, работа с элементами, ожидания)
- Кросс-браузерное тестирование (Chrome, Яндекс Браузер)
- Генерация отчётов Allure (вложение скриншотов, шаги с @Step)
- Работа с Git (ветки develop1, develop2, develop3, пул-реквесты)
- Сборка проектов в Maven (подключение зависимостей)

### 📌 Вывод

Проект охватил три ключевых уровня тестирования, что позволило продемонстрировать навыки от модульного до сквозного UI. Юнит-тесты обеспечили стабильность бизнес-логики бургера, API-тесты – корректность бэкенда, UI-тесты – пользовательские сценарии. Такой комплексный подход подтверждает готовность к реальной работе в качестве инженера по автоматизации тестирования.
